# Warband-to-Bannerlord Scene Converter

Warband-to-Bannerlord Scene Converter uses the output from
[Swyter's MaB Tools](https://github.com/Swyter/mab-tools) to convert Mount & Blade: Warband scene
data into files that are easier to import into Mount & Blade II: Bannerlord.

The tool converts terrain data, material maps, navmesh data, and scene entities. It also provides a
GUI for mapping Warband scene props to Bannerlord prefabs with position, rotation, scale, and origin
offset adjustments. Entity mappings are saved to `asset_mappings.xml`, so each converted scene can
make future conversions faster.

## Quick Start

1. Unpack the original Warband `.sco` scene file with `mab_sco_unpack.py` from
   [Swyter's MaB Tools](https://github.com/Swyter/mab-tools).
2. Run `WarbandToBannerlordConverter.exe`.
3. In **Unpacked Files Processor**, select the folder generated by `mab_sco_unpack.py`.
4. Click **Convert files**.
5. Import the generated terrain PNGs and navmesh prefab into a fresh Bannerlord scene.
6. Select the generated `mission_objects.json` and your Bannerlord `.xscene` file in the GUI.
7. Map Warband entities to Bannerlord prefab IDs.
8. Click **Inject All Into Scene**.

## Conversion Workflow

### 1. Unpack the Warband Scene

Use `mab_sco_unpack.py` from Swyter's MaB Tools to unpack the original Warband `.sco` scene file:

https://github.com/Swyter/mab-tools

The converter expects the unpacked folder produced by this tool.

### 2. Process the Unpacked Files

Run `WarbandToBannerlordConverter.exe`, select the unpacked folder in **Unpacked Files Processor**,
and click **Convert files**.

The converter writes generated files into that same folder:

- PNG heightmap from the Warband terrain elevation data
- PNG material maps from the Warband terrain layers
- Navmesh prefab `.bin`, if `ai_mesh.obj` exists

After conversion, note the displayed **Z Scale** and **Z Offset**. These values are needed when
importing the heightmap into Bannerlord.

### 3. Create the Bannerlord Scene

Create a fresh Bannerlord scene in the editor.

Set the terrain dimensions to match the original Warband scene. For native scenes and most mods, the
dimensions can be found in `scenes.txt`. For example:

```text
scn_random_scene_steppe random_scene_steppe 1792 none none 0.000000 0.000000 240.000000 240.000000 -0.500000 0x0000000229602800000691a400003efe00004b34000059be
  0
  0
 outer_terrain_steppe
```

In this example, the scene is `240 x 240` meters.

If you do not have the terrain code, you can infer the dimensions from the generated PNG size.
Warband commonly used polygon sizes of `2`, `3`, `4`, and `5`. Multiply the PNG dimensions by each
of those values to find the likely scene size.

For example, a `67 x 167` PNG could represent:

- `134 x 334`
- `201 x 501`
- `268 x 668`
- `335 x 835`

In that case, the original scene was likely intended to be around `200 x 500` meters. Some precision
is lost during conversion, so the inferred size may not be exact.

### 4. Import Terrain Data

In the Bannerlord editor:

1. Generate terrain using the original scene dimensions.
2. Import `layer_ground_elevation.png` as the heightmap.
3. Use the **Z Scale** and **Z Offset** values from the converter.
4. Create and select a new terrain paint layer.
5. Import each generated terrain layer PNG as a material map.
6. Assign the matching material name and texture for each layer.
7. Set **Water Level** to `0`.

You may need to reference the original Warband scene to determine the correct material layer order.

### 5. Import Navmesh Data

If a navmesh prefab was generated:

1. Move `(scene_name)_navmesh.bin` to:

   ```text
   steamapps\common\Mount & Blade II Bannerlord\Modules\(Your module name)\NavMeshPrefabs
   ```

2. In the Bannerlord editor, create an empty entity.
3. Set the empty entity's position and rotation values to `0`.
4. Name the empty entity the same as the navmesh `.bin` file.
5. Open **Navigation Mesh Panel > Generation Tools**.
6. Click **Import faces from prefabs**.
7. Enter the navmesh prefab name in both dialogs.

The navmesh should then be imported into the scene.

### 6. Map and Inject Scene Entities

Save the Bannerlord scene before injection.

In the converter GUI:

1. Set **Mission JSON** to the generated `mission_objects.json`.
2. Set **Scene XSCENE** to the fresh Bannerlord `.xscene` file.
3. Select each Warband entity in the left list.
4. Enter the matching Bannerlord prefab ID.
5. Adjust offsets, rotation, scale, and origin point as needed.
6. Click **Save Current Mapping** when needed.
7. Click **Inject All Into Scene**.

After injection, reopen the Bannerlord scene without saving over the injected changes first. The
inject button saves its changes automatically.

## GUI Reference

### Unpacked Files Processor

Select the folder created by `mab_sco_unpack.py`, then click **Convert files** to convert terrain
and navmesh data into Bannerlord-friendly files.

### Z Scale and Z Offset

Use these values when importing `layer_ground_elevation.png` as a Bannerlord heightmap.

### Mission JSON

Select the `mission_objects.json` generated by the unpacker. This file contains the Warband scene
entity list with original positions, rotations, and scales.

### Scene XSCENE

Select the Bannerlord `.xscene` file that should receive the converted entities. This file is edited
by the tool.

### Entity List

The left list shows all unique Warband scene entities that need a Bannerlord mapping.

### Bannerlord Prefab ID

The Bannerlord prefab or asset ID that should replace the selected Warband entity. Custom prefabs and
mod assets can be used as long as they are available in the Bannerlord editor.

### Global Position Offset

This should rarely be used. The offset is global, so an `X` offset moves every instance of that
mapping along the same world axis in every scene.

### Global Rotation Offset

Adjusts the Bannerlord entity rotation so it matches the mapped Warband entity.

### Global Scale Multiplier

Multiplies the converted scale of the Bannerlord prop. Use this to match the size of the Warband
counterpart. Negative scale values are supported, just like in Warband and Bannerlord.

### Origin Point Offset

Creates an empty parent entity for the selected mapping. This is useful when the Bannerlord prefab's
origin point does not match the Warband prop's origin point. With the correct origin offset, rotation
is applied around the expected point.

### Save Current Mapping

Manually saves the current mapping to `asset_mappings.xml`. The tool also autosaves changes.

### Inject All Into Scene

Writes all mapped entities into the selected `.xscene` file. Existing injected entities are removed
and replaced during injection.

## Notes for Users

- Consider adding meshes to empty entities to create missing prop equivalents. Bannerlord includes
  many meshes that are not available as ready-made prefabs.
- The Bannerlord prefab does not have to exactly match the original Warband prop. This can be useful
  when updating old scenes that previously worked around Warband asset limitations.
- More precise mappings reduce cleanup work after injection, but converted Warband scenes will still
  benefit from manual polish in the Bannerlord editor.

## Not Planned

### Automated Entity Mapping

Automated entity mapping may be technically possible, but it is outside the current scope of this
tool.

A possible approach would be to compare names semantically, generate shortlists of likely mappings,
then compare exported meshes from OpenBRF and TPAC tooling. That would be a much larger project than
this converter and would extend beyond Warband-to-Bannerlord scene conversion.
